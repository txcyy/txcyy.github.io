<h2 id="子模块">子模块</h2>
<h3 id="背景">背景</h3>
<p>有种情况我们经常会遇到：某个工作中的项目需要包含并使用另一个项目。 也许是第三方库，或者你独立开发的，用于多个父项目的库。 现在问题来了：你想要把它们当做两个独立的项目，同时又想在一个项目中使用另一个。</p>
<p>Git 通过子模块来解决这个问题。 子模块允许你将一个 Git 仓库作为另一个 Git 仓库的子目录。 它能让你将另一个仓库克隆到自己的项目中，同时还保持提交的独立。</p>
<h3 id="使用">使用</h3>
<h3 id="如何在现有项目中添加子模块">如何在现有项目中添加子模块</h3>
<p><code>git submodule add</code> 命令后面加上想要跟踪的项目的相对或绝对 URL 来添加新的子模块。</p>
<h3 id="如何克隆存在子模块的项目">如何克隆存在子模块的项目</h3>
<p>方式一</p>
<ol>
<li>
<p><code>git clone</code> 克隆项目</p>
</li>
<li>
<p><code>git submodule init</code> 用来初始化本地配置文件</p>
</li>
<li>
<p><code>git submodule update</code> 则从子项目中抓取所有数据并检出父项目中列出的合适的提交</p>
</li>
</ol>
<p>方式二</p>
<ol>
<li><code>git clone</code> 命令传递 <code>--recurse-submodules</code> 选项，它就会自动初始化并更新仓库中的每一个子模块， 包括可能存在的嵌套子模块。</li>
</ol>
<blockquote>
<p>如果你已经克隆了项目但忘记了 <code>--recurse-submodules</code>，那么可以运行 <code>git submodule update --init</code> 将 <code>git submodule init</code> 和 <code>git submodule update</code> 合并成一步。如果还要初始化、抓取并检出任何嵌套的子模块， 请使用简明的 <code>git submodule update --init --recursive</code>。</p>
</blockquote>
<h3 id="如何在父项目更新子项目">如何在父项目更新子项目</h3>
<ol>
<li>
<p>进入子项目手动更新</p>
</li>
<li>
<p><code>git submodule update --remote</code></p>
</li>
<li>
<p><code>git submodule update --init --recursive</code></p>
</li>
</ol>
<h3 id="如何在父项目发布子项目">如何在父项目发布子项目</h3>
<ol>
<li>
<p><code>git push --recurse-submodules=check</code> 在推送主项目之前检查子模块是否存在改动没有推送，如果是则主项目推送失败，需要手动进入子项目目录进行推送。</p>
</li>
<li>
<p><code>git push --recurse-submodules=on-demand</code> 在推送主项目之前自动推送子模块，如果子模块因为某些原因推送失败，主项目也会推送失败。</p>
</li>
</ol>
