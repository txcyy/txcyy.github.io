nextjs静态导出及部署

在 next.config.mjs 文件中配置 distDir 和 output 目录，执行next build命令即可。

/*next.config.mjs*/
const nextConfig = {
  distDir: process.env.NODE_ENV==='production'?'txcyy.github.io':'.next',
  ...(process.env.export?{output: 'export'}:{}),
  ...otherConfig
}

/*package.json*/
"scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start",
    "lint": "next lint",
    "export": "node movePostImgToPublic.js && set export=true && next build && start autoDeploy.bat"
  }

这里自定义了export命令，增加了一些发布相关的脚本。其中

movePostImgToPublic.js用于将.md文件中用到的图片移动到public目录

autoDeploy.bat用于进入到disDir对应的目录执行git相关命令，用于自动将构建出的静态页面push到github page项目上。

shadcn ui配置主题样式

shadcn提供Light、Dark两种主题。主题样式的配置放在components.json文件中，由于项目使用tailwindcss，因此在tailwind配置项中。

"tailwind": {
    "config": "tailwind.config.ts",
    "css": "src/app/globals.css",
    "baseColor": "slate",
    "cssVariables": true,
    "prefix": ""
  },

css：定义样式存放的文件，内容分别放在:root{} .dark{}两个选择器里，对应light、dark两种主题下需要使用的一系列css变量。

config：与tailwind结合预定义了一些自定义样式类等，例如定义primary颜色{primary: hsl(var(--primary))}， 那么就可以在页面中使用text-primary、bg-primary等类名，而tailwindcss原本是不存在text-primary这个样式类定义的。这样做相当于将变量的作用运用到了class类名上，从而使拥有相同类名的元素达到样式的统一及批量修改。

baseColor：slate是引入shadcn时选择的，选择之后不可修改，用于定义:root{} .dark{}里的css变量值。如果需要切换颜色，修改这里是没有作用的，需要直接修改:root{} .dark{}内的css变量值。

如果需要动态切换颜色，则需要动态添加颜色class，同时在globals.css文件中添加配套的CSS变量。例如需要添加blue颜色，那么需要动态设置<html class="blue">，然后在globals.css文件中添加配套变量:root.blue{} .dark.blue{}
