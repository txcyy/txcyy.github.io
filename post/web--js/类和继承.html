类和继承

类

类定义主要有以下几种方式：

  1. 工厂方式，存在语义上不像创建类实例（未使用new构造函数），每个实例都创建了方法的问题。
createA(name) {
    var a = new Object;
    a.name = name;
    a.getName = function(){};
    return a;
}
  1. 构造函数方式，存在每个实例都创建了方法的问题。
function A(name){
    this.name = name;
    this.getName = function(){}
}
  1. 原型方式，存在无法向构造函数传参，每个实例属性都一样的问题。
function A(){
}
A.prototype.name = 'a';
A.prototype.getName = function(){}
  1. 构造函数及原型混合方式，较为完美，但方法定义在构造函数外部。
function A(name){
    this.name = name;
}
A.prototype.getName = function(){}
  1. 动态原型方式，基本与混合方式一致，区别在于将方法定义在构造函数内。
function A(name){
    this.name = name;
    if(!A._inited) {
        A.prototype.getName = function(){}
        A._inited = true;
    }
}
  1. class方式,只是es6语法糖，内部基本与构造函数/原型缓和方式一致，constructor对应构造函数,static成员添加到构造函数，其余成员添加到构造函数的原型
class A{
    static a = 5
    constructor(name){
        this.name = name;
    }
    static getValue(){}
    getName(){}
}
// 等同于
function A(name){
    this.name = name;
}
A.a = 5;
A.getValue = function(){}
A.prototy.getName = funciton(){}

继承

javascript继承只能是单继承，可使用拷贝方式实现多继承效果。

  1. 混合方式
function Super(name) {
    this.name = name;
}
Super.prototype.getName = function(){
    return this.name;
}
function Sub(name, value) {
    Super.call(this,name);
    this.value = value;
}
// 此处不直接Sub.prototype = new Super()是因为这样的话Super的属性会附加在Sub.prototype上
function F(){}
F.prototype = Super.prototype;
Sub.prototype = new F()
Sub.prototype.constructor = Sub;
Sub.prototype.getValue = function(){
    return value;
}
  1. object.create方式 基本与混合方式一致，区别在于无需使用中间的构造函数F，而是使用Object.create()：
function F(){}
F.prototype = Super.prototype;
Sub.prototype = new F()
// 以上代码替换为
Sub.prototype = Object.create(Super.prototype)
  1. 拷贝方式
Sub.prototype = Object.assign({},Super.prototype,A.prototype,B.prototype,{
    constructor:Sub,
    otherProp: '',
    otherFunction: function(){}
})
  1. class extends方式，只是es6语法糖，内部实现基本与object.create方式一致,super相当于A.call(this)
class A{
    constructor(){
        
    }
}
class B extend A{
    constructor(){
     super()   
    }
}
